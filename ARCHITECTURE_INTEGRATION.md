## Источники (проверено на реальных данных)

### 1. ~~DWH ClickHouse `f_cloud` — ФАКТЫ~~ (УДАЛЕНО — см. баннер выше)
- `org` — аптеки. Ключ аптеки = **`fomid`** (где `apt=True`, `fomid>0`). `inn` грязный (переиспользуется как контрагент), поэтому фильтруем `apt=True AND fomid>0`.
- `good` — товары. Поле **`fom_good`** = глобальный FOM ID товара (= «FOM ID» в Проекты.xlsx). Таблица 8М строк → **запрашивать только с фильтром `pharmacy_id`** (у каждой аптеки свой набор).
- `incomeln` — приходы (ЗАКУП): `good`(=good.id), `kol`, `priceprih`, `data`, `pharmacy_id`.
- `invoiceln`/`invoice` — продажи (ПРОДАЖА): по тому же `pharmacy_id`.

### 2. Проекты.xlsx (лист «Лист1», ~995 товаров, 19 проектов) — СПРАВОЧНИК ТОВАР→ПРОЕКТ+ЦЕНЫ
| Кол | Поле | Назначение |
|---|---|---|
| C | FOM ID | ключ товара → DWH `good.fom_good` |
| D | Наименование ЛС | название |
| E | Проект | проект (KRKA, Getz, Bayer…) |
| F | CIP цена | закуп-цена (для маржи) |
| K | Бонус Аптека закуп | бонус аптеки при закупе |
| L | Бонус Аптека продажа | бонус аптеки при продаже |
| I | Менеджер | менеджер проекта |

### 3. СВОД ...xlsx — ПЛАНЫ
- Лист **«II-Q»** (матрица аптеки × проекты): стр.3 шапка (B=ИНН, D=DATFO INN, F=юр.название, G=сеть, I=кол-во аптек, J=менеджер, K=регион, L=район), далее блоки план/факт по проектам.
- Лист **«Свод таб »** (формат одной аптеки): Проект / Статус / **Условия (Закуп/Продажа)** / План+Факт по месяцам. Совпадает с форматом текущего `dashboard_sync.py`.

## Ключевая логика
- **Один ИНН = одна точка (fomid).** Агрегация сети не нужна.
- **Условие проекта (Закуп/Продажа)** выбирает источник факта:
  - «Закуп» → факт = Σ `incomeln` (priceprih×kol) товаров проекта.
  - «Продажа» → факт = Σ `invoiceln` товаров проекта.
- Период = **II Q (апрель, май, июнь 2026)**.

## Поток ETL (на каждую аптеку, всё под лимит 1М строк)
```
inn (из планов)
  └─► org: SELECT fomid FROM org WHERE inn=? AND apt=1 AND fomid>0
        └─► good: SELECT id, fom_good FROM good WHERE pharmacy_id=fomid     (карта good.id→fom_good)
              ├─► incomeln: Σ по good.id за месяц/квартал  (ЗАКУП)
              └─► invoiceln: Σ по good.id за месяц/квартал (ПРОДАЖА)
                    └─► маппинг fom_good→проект (Проекты.xlsx)
                          └─► факт проекта = Σ по условию (Закуп/Продажа)
                                └─► план проекта (СВОД II-Q) → % = факт/план
                                      └─► бонус (Проекты K/L при достижении плана)
                                            └─► dashboard_data → PostgreSQL pharmacies (по inn)
```

## Хранение
- Витрина: PostgreSQL `pharmacies.dashboard_data` (как сейчас) + опц. таблицы `products`/`pharmacy_products` для экрана товаров.
- Синк по расписанию (ночью). App читает только PostgreSQL — DWH не нагружаем.
