# Спецификация данных для интеграции DATFO (Excel)

> **СТАТУС (обновлено 2026-06-29): интеграция с DWH ClickHouse остановлена.**
> Везде ниже, где написано «нужно отдать из DWH», источником теперь является **Excel**:
> `data/plans.xlsx` (лист «II-Q» — планы и факты) и `data/projects.xlsx` (справочник товаров,
> цены CIP, ставки бонусов). Прямой доступ к ClickHouse больше не используется и не планируется.

Документ описывает, какие данные нужны продукту, чтобы он полностью работал на реальных данных
(Telegram Mini App + мобильное приложение используют один backend).

**Главный ключ связи всего — ИНН аптеки** (`inn`). По нему связываются аптека, её показатели,
проекты и товары. Пользователь (Telegram или логин) тоже привязан к одному/нескольким ИНН.

Легенда статуса:
- ✅ **есть** — уже приходит (Google Sheets «Свод таб new»)
- ⭐ **НУЖНО** — сейчас фейковое/отсутствует, нужно отдать из Excel (plans.xlsx / projects.xlsx)

---

## 1. Доступ: кто какие аптеки видит

| Поле | Тип | Назначение |
|---|---|---|
| `inn` | строка | Ключ аптеки |
| `telegram_id` | число | Telegram-владелец аптеки (если есть). ✅ есть (мастер-лист) |
| `login` / роль | — | Для входа вне Telegram заводим мы; нужен лишь список ИНН на пользователя |
| `role` | user / admin / superadmin | Обычный видит свои ИНН, админ — все |

Нужно от вас: **соответствие `telegram_id → ИНН`** (и кто админ).

## 9. ⭐⭐ ТОВАРЫ по каждому проекту — ГЛАВНЫЙ НЕДОСТАЮЩИЙ БЛОК

Сейчас это полностью фейк (`FAKE_PRODUCTS`, случайные статусы/объёмы/«соседи»).
Чтобы экран «продукция проекта» (см. скриншот: Нолипрел Би-форте…) показывал правду,
нужны данные на уровне **аптека × товар**.

### 9.1 Справочник товаров проекта (общий, без привязки к аптеке)

| Поле | Пример | Обяз. |
|---|---|---|
| `product_id` | `krka_noliprel_5125` | ✅ ключ |
| `project_name` | `KRKA` | ✅ |
| `name` | `Нолипрел Би-форте` | ✅ |
| `dosage` | `5+1.25 мг` | ✅ |
| `form` | `таб. №30` | желательно |
| `price` | `95000` | ✅ закупочная цена для аптеки |
| `retail` | `145000` | ✅ рекомендуемая розница (маржу считаем сами) |
| `bonus_pct` | `7` | ✅ % бонуса по товару |

### 9.2 Закупка товара конкретной аптекой (аптека × товар)

| Поле | Пример | Назначение |
|---|---|---|
| `inn` | `305181792` | ✅ ключ аптеки |
| `product_id` | `krka_noliprel_5125` | ✅ ключ товара |
| `status` | `active` / `occasional` / `missed` | покупает / пробовал / не покупает |
| `monthly_orders` | `11` | среднемес. объём закупки (упак/мес) |
| `potential` | `336840` | (опц.) потенциальный бонус за квартал; если не дадите — посчитаем |

### 9.3 (Опционально) «Давление на допродажу» — конкурентная справка
Используется, чтобы мотивировать аптеку («соседи берут, а вы упускаете»). Если нет — скроем.

| Поле | Пример |
|---|---|
| `peers_total` | `12` (аптек в регионе) |
| `peers_buying` | `8` (берут этот товар) |
| `competitor` | `Лористу` (конкурентный товар) |
| `competitor_brand` | `Berlin-Chemie` |
| `competitor_reason` | `Дороже для аптеки, маржа ниже` |

## Формат и способ передачи

Подойдёт любой из вариантов (по убыванию удобства):
1. **Прямой доступ к таблицам/вью в DWH** (или реплике) — мы читаем по расписанию.
2. **Выгрузка CSV/Excel** по каждому блоку (аптеки, проекты, товары-справочник, закупки-аптека×товар).
3. **Google Sheets** (как сейчас) — продолжаем текущий пайплайн.

Уточнить:
- **Как часто обновлять** (квартал / месяц / ежедневно).
- **Единицы и формат чисел** (лучше сырые числа, валюта — сум).
- **Ключи**: `inn` (аптека), `product_id` (товар), `project_name` (проект).

## Что критично для «фулл в продукт»
1. **`telegram_id → ИНН`** + кто админ (раздел 1).
2. **Товары проекта** (раздел 9) — без них экран продукции остаётся фейковым.
3. **Менеджеры** (раздел 3) — сейчас часто пусто, а на них завязаны кнопки «связаться».

Разделы 2,4–8 уже работают из Google Sheets — нужно лишь подтвердить полноту/актуальность.
