============================================================
DATFO — ПОЛНЫЙ ФУНКЦИОНАЛ, ФАЙЛЫ И API
(сценарий: в системе НЕТ данных — что нужно, чтобы всё заработало с нуля)
============================================================

Backend обслуживает Telegram Mini App (бот + веб-апп).
Главный ключ связи всего — ИНН аптеки (inn).

ЛЕГЕНДА:
  [ЕСТЬ]  — уже реализовано / приходит из Google Sheets
  [НУЖНО] — отсутствует или фейковое, надо подключить реальные данные/код


============================================================
0. АРХИТЕКТУРА
============================================================
- bot.py              Telegram-бот (aiogram), кнопка Web App, рассылки, опросы.
- api.py              HTTP API (aiohttp) + раздача webapp/. Стартует из bot.py.
- database.py         Доступ к PostgreSQL, схема таблиц.
- dashboard_sync.py   Синк показателей аптек из Google Sheets -> dashboard_data.
- seed_clients.py     Синк связки ИНН <-> telegram_id из мастер-листа.
- create_app_user.py  Создание логин/пароль пользователей (вход вне Telegram).
- webapp/             Фронтенд (index.html, app.js, config.js, assets/).

Поток: Excel/Sheets -> синк-скрипты -> PostgreSQL -> api.py -> webapp (Telegram Mini App).

3. API ЭНДПОИНТЫ (контракт)
============================================================
Авторизация во ВСЕХ защищённых ручках — одним из:
  - Telegram: заголовок X-Telegram-Init-Data (или ?init_data=) — проверка подписи
  - Приложение: заголовок Authorization: Bearer <token> (получен на /api/auth/login)
  - Dev: ?tg_id=<id> (только при ALLOW_QUERY_TG_ID=1)

[ЕСТЬ] POST /api/auth/login
        in:  {login, password}
        out: {token, login, is_admin}
        данные: app_users, app_user_inns

[ЕСТЬ] POST /api/auth/logout
        in:  заголовок с токеном
        out: {ok:true}

[ЕСТЬ] GET /api/me
        Профиль + список доступных аптек (для админа — все, slim).
        out: {tg_id, login, role, is_admin, language, pharmacies[], managers[]}
        данные: users/app_users, pharmacies(+dashboard_data)

[ЕСТЬ] GET /api/pharmacy/{inn}
        Полные данные одной аптеки (дашборд) с проверкой доступа.
        out: {id, inn, name, business, dashboard{...}}
        данные: pharmacies.dashboard_data

[ЕСТЬ] GET /api/health
        Проверка живости. out: {ok:true}

[ЕСТЬ] POST /api/events
        Трекинг кликов/действий (аналитика).
        in:  {event, pharmacy_inn, payload}
        данные: events

[ЕСТЬ] POST /api/ai/ask
        AI-ассистент по данным аптеки + базе знаний.
        in:  {question}
        out: {answer, model}
        данные: dashboard_data + база знаний; ключ ANTHROPIC_API_KEY

[ЕСТЬ] GET /api/admin/stats?days=N
        Сводка активности (только admin). данные: events

[ЕСТЬ] GET /            раздача webapp/index.html + статика (config.js, app.js, assets)

[НУЖНО] GET /api/products?inn=<inn>&project=<name>
        Реальные товары проекта для аптеки (заменяет фейковый генератор во фронте).
        out: [{product_id, name, dosage, form, price, retail, bonus_pct,
               status, monthly_orders, potential, peer{...}}]
        данные: products + pharmacy_products
        АЛЬТЕРНАТИВА: вкладывать products прямо в dashboard_data.projects[].products[]
                      и отдавать через /api/pharmacy/{inn} (без нового эндпоинта).

6. ПРИОРИТЕТ ДЛЯ ЗАПУСКА «С НУЛЯ»
============================================================
1) PostgreSQL + database.py create_tables()           -> пустые таблицы
2) Аптеки + связка telegram_id<->ИНН (seed_clients)    -> кто что видит
3) Показатели аптек (dashboard_sync)                   -> дашборд оживает
4) Менеджеры заполнены                                 -> кнопки «связаться»
5) ТОВАРЫ: products + pharmacy_products + products_sync -> экран продукции на реальных данных
6) WEB_APP_URL (постоянный домен) + бот запущен        -> Mini App в Telegram
7) (опц.) ANTHROPIC_API_KEY                            -> AI-ассистент
8) (опц.) база знаний                                  -> FAQ/AI ответы

КРИТИЧНО, что сейчас фейковое/пустое:
  - товары проекта (раздел 2.9 / 2.10) — генерируются случайно во фронте;
  - связка telegram_id -> ИНН и роли;
  - менеджеры по аптекам.
Остальное (аптеки, квартал, проекты, бонусы, месяцы) уже работает из Google Sheets.
